Fundamentals of HTML

HTML, or HyperText Markup Language, is the cornerstone of web development, serving as the foundation for creating web pages and applications. It allows developers to structure content with a range of elements and tags, such as p for paragraphs, a for links, and img for images.

These elements tell web browsers how to display content, enabling the creation of diverse and interactive web experiences. HTML is not a programming language but a markup language, emphasizing the importance of properly structuring and categorizing content to ensure accessibility and search engine optimization (SEO).

Structure & Elements

HTML documents are structured using a nested hierarchy of elements, starting with the html element that encapsulates the entire document. Within this, the head section contains meta-information like the title and links to CSS files, while the body section hosts the content visible to users.

Elements, represented by tags, define different parts of a webpage, such as headings (h1 through h6), paragraphs (p), and lists (ul, ol). This structure is fundamental for creating organized and accessible web content, allowing browsers to render pages correctly.

Element Description
HTML This is the root element of an HTML page. It wraps all the content on the entire page. It is sometimes known as the root element.
Head A container for all the head elements relating to metadata. These typically include title, link, meta, script, and base.
  • Title: Displays in the tab of the browser as the title of the page, and is userful for SEO purposes.
  • Link: Used to link to external style sheets.
  • Meta: Contains metadata about the HTML document, such as the character set, page description, keywords, author of the document, and more.
  • Script: Used to define client-side JavaScript.
  • Base: Used to specify the base URL and target for all relative URLs in a page.
Body Contains all the content of an HTML document, such as text, hyperlinks, images, tables, lists, etc.
  • Header: Contains introductory content or navigational aids for the page.
  • Nav: Contains a collection of navigational links.
  • Main: Contains the main content of the page, such as articles, blog posts, etc.
  • Section: Contains related content grouped together, such as blog posts, articles, etc.
  • Article: Contains independent, self-contained content, such as blog posts, articles, etc.
  • Aside: Contains content that is tangentially related to the content around it, such as sidebars, pull quotes, etc.
  • Footer: Contains the footer of the page, such as copyright information, contact information, etc.

Semantic HTML

Semantic HTML involves using specific HTML tags that provide meaning to the web content, helping both browsers and developers understand the purpose and structure of web documents. Unlike stylistic uses of elements (e.g., using b for bold text), semantic elements like article, aside, footer, and nav describe the role of the content in the document.

This practice improves SEO, accessibility, and maintainability, as it makes the structure of web pages more logical and easier to interpret by assistive technologies and search engines.

Tables

HTML tables are a way to display data in a grid format, composed of rows and columns. Created with the table element, tables are structured using tr for table rows, td for table data cells, and th for header cells that define column or row headings. Tables can include attributes to control spacing, border thickness, and alignment, making them versatile for displaying anything from simple lists to complex data sets. However, for non-tabular data or complex layouts, CSS-based layouts are recommended for better accessibility and flexibility.

Fundamentals of CSS

CSS, or Cascading Style Sheets, is a powerful tool that defines the look and feel of HTML content on the web. By separating content (HTML) from presentation (CSS), developers can achieve a high degree of control over layout, colors, fonts, and responsiveness across different devices. CSS syntax consists of selectors (identifying the HTML elements to style) and declarations (defining how the elements should be styled), all contained within style sheets. This separation of concerns not only makes websites visually engaging but also enhances maintainability and reusability of code.

Selectors & Visual Rules

CSS selectors are patterns used to select the HTML elements you want to style. From simple selectors that target elements by type, ID, or class, to complex selectors that apply styles based on an element's state or its relationships with other elements, selectors are fundamental to applying CSS styles.

Visual rules, defined within curly braces after the selector, specify the styles to be applied, such as color, font size, margin, and padding. Understanding CSS selectors and how to effectively apply visual rules is crucial for creating visually compelling and structurally sound web pages.

Box Model

The CSS Box Model is a conceptual framework that describes how every element on a web page is rendered as a rectangular box. This model encompasses margins (space outside the border), borders (the outer edge of the element), padding (space between the border and the content), and the content area itself.

The box model is fundamental in CSS layout, as it determines how elements interact with each other and how much space they occupy. Understanding and manipulating the box model is essential for precise control over layout and spacing in web design.

Display & Positioning

In CSS, the display and positioning properties determine how elements are laid out on the page. The display property specifies whether an element is treated as a block (occupying its own line), inline (flowing with text), or other formats like flex and grid that provide advanced layout capabilities.

The position property, on the other hand, defines how an element is positioned within its parent container. Elements can be positioned statically (the default), relatively (relative to its normal position), absolutely (relative to the nearest positioned ancestor), or fixed (relative to the viewport).

Understanding these properties is crucial for creating responsive and visually appealing web layouts.
